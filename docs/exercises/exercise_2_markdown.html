Exercise 2 — Write Your README with Markdown

Module 1 | ~20 minutes

Goal

Edit your GitHub profile README using Markdown formatting.

Steps

Open the README editor

  1. Go to your repository: https://github.com/YOUR_USERNAME/YOUR_USERNAME
  2. Click the pencil icon on the README.md file to edit it

Write your introduction

  1. Write at least the following sections using Markdown:
# Your Name

## About Me
One or two sentences about your role and research.

## Research Project
Brief description of what you are currently working on.

## Affiliation
- Institution
- Research group / lab

## Education

| Degree | University | Year |
|--------|-----------|------|
| PhD    | UZH       | 2024 |
| MSc    | ...       | ...  |

## Links
- [LinkedIn](https://linkedin.com/in/your-profile)

Commit the file

  1. Scroll down to Commit changes
  2. Write a meaningful commit message: added introduction and education table
  3. Click Commit changes

Compare

  1. Go to https://github.com/YOUR_USERNAME
  2. Compare your profile page to the screenshot from Exercise 1 — what changed?

Check your understanding

Question: You wrote # Your Name at the top of the file. What would happen if you used ### instead of #?

TipClick to reveal the answer

It would still render as a heading, but a much smaller one (Heading 3 instead of Heading 1). The number of # symbols sets the heading level — more # = smaller text. This matters for document structure: search engines and screen readers use heading levels to understand a page’s outline, not just its appearance.

Question: Your education table used | to separate columns. What happens if a row has fewer | separators than the header row?

TipClick to reveal the answer

The table breaks — Markdown table renderers expect every row to have the same number of columns as the header. A missing | usually causes the remaining cells in that row to merge into the last column, or the whole table to fail to render as a table at all. Always count your | characters when copy-pasting table rows.

Markdown you practiced

Syntax Result
# text Heading 1
## text Heading 2
**text** bold
- item bullet list
| col | table column
